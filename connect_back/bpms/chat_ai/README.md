# Chat AI

Приложение **chat_ai** — это модуль для организации диалога пользователя с AI-ботом, который автоматически распознаёт намерения (intents) и формирует структурированные данные для дальнейшей обработки.

## Основные сущности

- **AIChatModel** — чат.  
  Хранит автора, название и время последнего сообщения.  
  Если у пользователя нет активных чатов, при первом запросе создаётся новый.

- **AIMessageModel** — сообщение в чате.  
  Содержит текст, автора (пользователь или бот), ссылку на предыдущее сообщение и флаг, является ли сообщение ботом/намерением.

- **IntentTypeModel** — тип намерения.  
  Определяется JSON-схемой (meta), в которой описаны поля (обязательные, типы, модели-ссылки и т. п.).

- **IntentModel** — конкретное намерение, связанное с сообщением.  
  Содержит исходные данные (raw_data), карту сопоставлений (resolutions), готовые данные для сериализаторов (resolved_data), статус и связанные ошибки.

- **IntentStatusModel** — справочник статусов намерений (`resolving`, `ready`, `done`, `failed`).

## API эндпоинты

Базовый префикс:  
```
/api/v1/chat_ai/
```

### 1. Получить список чатов
Если у пользователя нет ни одного активного чата — создаётся новый.

```
GET /api/v1/chat_ai/chats/
```

### 2. Получить список сообщений чата
Указывается `chat` в query-параметре.

```
GET /api/v1/chat_ai/messages/?chat=<chat_id>
```

### 3. Отправить сообщение в чат
Пользователь отправляет текст. В ответ возвращается сообщение-ответ от бота (с выявленными намерениями).

```
POST /api/v1/chat_ai/messages/
Content-Type: application/json

{
  "chat": "27634ec2-8ef2-11f0-90b7-8d510b2412da",
  "text": "Создай задачу"
}
```

Ответ: JSON с сообщением бота и вложенными Intent’ами.

### 4. Уточнение значения для поля намерения
Позволяет выбрать кандидат или ввести значение вручную.  
Метод: `PUT /intents/{intent_id}/update-value/`

Примеры:

- Текстовое поле:
```json
{
  "field_name": "name",
  "value": "Подготовить презентацию"
}
```

- Дата:
```json
{
  "field_name": "date_start_plan",
  "value": "2025-03-20T15:00"
}
```

- ForeignKey (один id):
```json
{
  "field_name": "operator",
  "value": "a7745176-7e6f-11f0-8b37-4216f3de51df"
}
```

- ManyToMany (список id):
```json
{
  "field_name": "visors",
  "value": [
    "92820932-bcf7-11ef-890e-4216f3de51df",
    "a7745176-7e6f-11f0-8b37-4216f3de51df"
  ]
}
```

### 5. Удалить чат
Деактивирует чат, устанавливая флаг `is_active` в `false`.

```
POST /api/v1/table_actions/update_is_active/
{
  "id": "20064af2-8fa7-11f0-93de-4216f3de51df",
  "is_active": false
}
```

## Логика обработки намерений

1. **raw_data** (от AI) — исходный JSON.  
2. **resolutions** — промежуточная карта, где указаны кандидаты для ссылочных полей, текущее значение и статус.  
3. **resolved_data** — итоговый payload для бизнес-логики.  
4. **status**:
   - `resolving` — требуется уточнение пользователя,
   - `ready` — все обязательные поля определены.
   - `done` — создан связанный объект, ссылка на него в related_object.
   - `failed` — не удалось создать  связанный объект.
