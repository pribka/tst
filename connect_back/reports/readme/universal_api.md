# 📘 Инструкция по использованию UniversalModelViewSet API

Поддержка:
- Фильтрации (`filters`)
- Ограничения полей (`fields.fields_only`)
- Агрегаций (`fields.aggregates`)
- Группировки (`fields.groups`)
- Сортировки (`ordering`)
- Пагинации
- Мета-информации (`meta=true`)

---

## 🔗 Базовые эндпоинты

| Метод | URL                                | Описание                       |
|-------|------------------------------------|--------------------------------|
| GET   | `/api/<model_name>/`               | Получение списка объектов      |
| GET   | `/api/<model_name>/?meta=true`     | Получение мета-информации     |
| GET   | `/api/<model_name>/<id>/`          | Детали объекта                 |
| POST  | `/api/<model_name>/`               | Создание объекта               |
| PATCH | `/api/<model_name>/<id>/`          | Частичное обновление объекта  |

---

## 📦 Получение мета-описания модели

```http
GET /api/<model_name>/?meta=true
```

### Ответ содержит:
- `fields` — список полей и их описание
- `forms` — описание UI-форм
- `user_rights` — доступные действия пользователю
- `tps` — табличные части

## 🔍 Фильтрация

### Параметр: `filters=<JSON-массив>`

```json
[
  {
    "field": "year",
    "comparison_type": ">=",
    "value": 2020
  },
  {
    "field": "book_author__name",
    "comparison_type": "icontains",
    "value": "Толстой"
  }
]
```

```json
{
  "logic": "and",
  "filters": [
    { "field": "status", "comparison_type": "=", "value": "активен" },
    {
      "logic": "or",
      "filters": [
        { "field": "priority", "comparison_type": "=", "value": "высокий" },
        { "field": "priority", "comparison_type": "=", "value": "средний" }
      ]
    }
  ]
}
```


### Поддерживаемые `comparison_type`

| Тип             | Описание                         |
|-----------------|----------------------------------|
| `=`             | Равно                            |
| `!=`            | Не равно                         |
| `>`             | Больше                           |
| `>=`            | Больше или равно                 |
| `<`             | Меньше                           |
| `<=`            | Меньше или равно                 |
| `icontains`     | Содержит подстроку (без учёта регистра)   |
| `not icontains` | Не содержит подстроку (без учёта регистра)   |
| `isnull`        | Проверка на null (`true`/`false`)|
| `in`            | В списке (`строка через запятую` или массив) |
| `not in`        | Не в списке                      |

### ✅ **Разыменование ForeignKey осуществляется через двойное подчеркивание** (`__`), как в Django:
> Например: `book_author__name`, `user__profile__department__title`

### Пример:
```
/api/BookModel/?filters=[{"field":"year","comparison_type":">=","value":2020}]
```

### Далее по тексту описана группировка и агрегация. Поля группировок и агрегаций так же могут участвовать в фильтрах

---

## 📊 Агрегации и группировка

### Параметр: `fields=<JSON>`

```json
{
  "groups": [
    {"name": "book_author__name", "order": 1, "is_visible": true},
    {"name": "category", "order": 2, "is_visible": false}
  ],
  "aggregates": [
    {"name": "avg_rating", "avg": "rating", "order": 3, "is_visible": true},
    {"name": "book_count", "count": "id", "order": 4, "is_visible": false}
  ]
}
```

| Ключ             | Описание                                       |
|------------------|------------------------------------------------|
| `sum`            | Сумма (`Sum`)                                  |
| `avg`            | Среднее значение (`Avg`)                       |
| `min`            | Минимум (`Min`)                                |
| `max`            | Максимум (`Max`)                               |
| `count`          | Количество (`Count`)                           |
| `distinct_count` | Количество уникальных (`Count`, distinct=True) |
| `concatenate`    | Объединение текстовых строк (`StringAgg`)      |
| `order`          | Порядок отображения столбца (число)            |
| `is_visible`     | Видимость в отчете (`true`/`false`, по умолч. `true`) |

### ✅ **В `groups` и `aggregates` также используется разыменование через `__`**
Если передать ManyToMany поле в `groups`, то количество записей увеличится на количество связанных объектов.

### 👁️ **Управление видимостью полей**
Параметр `is_visible` позволяет скрывать поля в финальном отчете, при этом они продолжают участвовать во всех вычислениях (группировка, агрегация). Полезно для скрытия технических полей или промежуточных вычислений.

### Пример с count:
```
/api/BookModel/?fields={"groups":["book_author__name"],"aggregates":[{"name":"total","count":"id"}]}
```

### Пример с concatenate:
```
/api/BookModel/?fields={"groups":["author"],"aggregates":[{"name":"total_description","concatenate":"description", "delimiter":". "}]}
```
Для варианта concatenate можно дополнительно передать разделитель (delimiter). По умолчанию используется ", ".

## 📃 Ограничение состава полей (поле `fields_only` в `fields`)

Можно ограничить список возвращаемых полей:

```json
{
  "fields_only": ["name", "year", "book_author__name"],
}
```
`fields_only` нельзя передавать с groups или aggregates.
ManyToMany поля отображаются как список сериализованных объектов.

## 📄 Сортировка

### Параметр: `ordering`

```text
ordering=year,-book_author__name
```

- `-` перед полем — сортировка по убыванию
- ✅ **Поддерживается разыменование через `__`**

---

## 📑 Пагинация

По умолчанию используется `LimitOffsetPagination`.

| Параметр | Назначение                          |
|----------|--------------------------------------|
| `limit`  | Сколько объектов вернуть (по умолч. 20, макс.2000) |
| `offset` | С какого объекта начать              |

### Пример:
```
/api/BookModel/?limit=10&offset=20
```

Для `PageNumberPagination` необходимо передать get-параметр `pagination=page`.

| Параметр | Назначение                          |
|----------|--------------------------------------|
| `page_size`  | Сколько объектов вернуть (по умолч. 20, макс.2000) |
| `page` | Номер страницы              |

### Пример:
```
/api/BookModel/?pagination=page&page=1&page_size=15
```

---

## 🧪 Комбинированный пример

```http
GET /api/BookModel/?
  fields={
    "groups": ["book_author__name"],
    "aggregates": [{"name":"avg_rating","avg":"rating"}]
  }
  &filters=[
    {"field":"year","comparison_type":">=","value":2010}
  ]
  &ordering=-avg_rating
  &limit=10
```

## 🌍 Настройка часового пояса

### Параметр: `timezone`

Позволяет указать часовой пояс пользователя в формате IANA для корректного преобразования столбцов типа DateTime:

```json
{
  "timezone": "Europe/Moscow"
}
```

### Примеры часовых поясов:
- `"Europe/Moscow"` — Московское время
- `"Asia/Almaty"` — Алматы
- `"America/New_York"` — Восточное время США
- `"UTC"` — Всемирное координированное время

Эта настройка влияет на отображение всех полей типа DateTime в ответе API, конвертируя их из UTC в указанный часовой пояс. Если timezone не передан, то по умолчанию берется часовой пояс сервера (TIME_ZONE в settings).

## ⚙ Примечания

- ForeignKey-поля сериализуются в виде:
  ```json
  {
    "id": "uuid",
    "repr": "Иван Иванов",
    "ct": { ... }
  }
  ```
- ManyToMany-поля сериализуются как массив ID:
  ```json
  [ "uuid1", "uuid2", "uuid3" ]
  ```
- Все поля можно управлять с фронта через `filters`, `fields`, `ordering`.

## 📝 Дополнительные параметры экспорта

### `template`

Позволяет использовать заранее подготовленный Excel-шаблон для экспорта отчёта. В шаблоне можно оформить шапку, футер, стили, ширину колонок и т.д. Передайте UID файла-шаблона (модели File) в параметре `template`:

```json
{
  "template": "<uuid-шаблона>"
}
```

- Все данные будут вставлены в первую страницу шаблона.
- Поддерживаются именованные диапазоны `header`, `footer`, `filter`, `table` для гибкой разметки.
- Плейсхолдеры `{{ report_title }}` и `{{ creation_date }}` автоматически заменяются на название отчёта и дату создания.
- Если фильтров или данных больше, чем предусмотрено в шаблоне, таблица автоматически сдвигается вниз.
- В качестве шаблона можно использовать обычный файл с расширением xlsx. 

### `grouping_fields_layout`

Определяет способ отображения иерархии группировок в экспорте (Excel/HTML):

- `together` (по умолчанию): групповые поля выводятся вертикальным стеком в первом столбце, остальные — обычными колонками. Удобно для вложенных иерархий.
- `separate`: каждое поле группировки — отдельная колонка, без вертикального стека.
- `separate_in_totals_only`: групповые поля скрыты в строках данных, но отображаются в строках итогов.

Пример:

```json
{
  "grouping_fields_layout": "separate"
}
```

## 🗑️ Очистка кэша мета-данных

### `POST /api/v1/reports/clear-meta-cache/`

Очищает кэш мета-данных для всех моделей из `REPORTS_UNIVERSAL_MODELS`. Требует права администратора. На случай изменения meta_exclude_fields, чтобы не чистить весь кэш.
