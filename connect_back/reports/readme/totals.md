
# 📊 Поддержка общих итогов в Excel-отчётах (`totals_param`)

## 📌 Описание

Система экспорта отчётов в Excel поддерживает вывод **общей строки итогов** (например, "ИТОГО") внизу таблицы.  
Поведение управляется через GET-параметр `totals`, который должен содержать словарь с ключом `grand_totals`.

## 🧾 Формат GET-параметра `totals`

```json
totals = {
  "grand_totals": {
    "name": "totals",
    "by": "columns",
    "title": "ИТОГО",
    "position": "bottom",
    "fields": ["cost", "duration"]
  }
}
```

Передаётся в URL как строка, например:

```
?totals={"grand_totals":{"by":"columns","title":"ИТОГО","position":"bottom","fields":["cost","duration"]}}
```

---

## ⚙️ Поддерживаемые параметры

| Параметр     | Тип       | Обязательный | Описание |
|--------------|-----------|--------------|----------|
| `name`       | `str`     | нет          | Название настройки. Сейчас не используется, зарезервировано. |
| `by`         | `str`     | да           | Тип агрегации: <br>✅ `"columns"` — по столбцам. <br>🚫 Другие значения пока игнорируются. |
| `title`      | `str`     | нет          | Подпись строки итогов. По умолчанию `"ВСЕГО"`. |
| `position`   | `str/null`| нет          | Где разместить строку итогов: <br>✅ `"bottom"` — внизу таблицы.<br>🚫 `"top"` и другие — пока игнорируются.<br>🛑 `null` или отсутствие — **итоги не выводятся**. |
| `fields`     | `list`    | нет          | Список полей, по которым нужно считать итоги. Если не указан — используются все `numeric_aggregate_fields`. |

---

## 🔍 Примеры использования

### ✔️ Стандартный запрос с кастомной подписью и выбором полей
```
?totals={"grand_totals":{"by":"columns","position":"bottom","title":"ИТОГО","fields":["cost","duration"]}}
```

### ✔️ Вывести итоги по всем числовым полям (если `fields` не указан)
```
?totals={"grand_totals":{"by":"columns","position":"bottom","title":"ВСЕГО"}}
```

### 🛑 Отключить вывод общих итогов
```
?totals={"grand_totals":{"position":null}}
```

---

## 📎 Поведение по умолчанию (если параметр `totals` не передан)

- Итоги **всегда выводятся внизу таблицы**.
- Используются **все поля из `numeric_aggregate_fields`**.
- Подпись — `"ВСЕГО"`.

---

## 🚫 Ограничения

- Поддерживается только `position = "bottom"` и `by = "columns"`.
- Параметр должен быть корректным JSON.
- Ошибки в формате параметра (`totals`) игнорируются, и итоговая строка ведёт себя как по умолчанию.
