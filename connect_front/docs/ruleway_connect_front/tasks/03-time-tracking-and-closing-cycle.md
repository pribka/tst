# Задания По Блоку: Трудозатраты, Закрытие Недели И Отчетность

## Цель

Сделать из трудозатрат и закрытия недели не побочный процесс, а обязательный и понятный пользовательский контур.

## Задания

1. Реализовать единый фронтовый worklog-компонент.
   Должен работать для задач, обращений, встреч, заявок и иных объектов.

2. Сделать ежедневный режим фиксации трудозатрат.
   Нужны: быстрый ввод, предзаполнение контекстом, подсказки по незалогированным работам.

3. Собрать `week closing wizard`.
   Шаги: незавершенные задачи, просрочки, висяки, незаполненные трудозатраты, неподтвержденные итоги.

4. Добавить расхождения между операционным и отчетным контуром.
   Примеры: задача без часов, часы без объекта, закрытая неделя с просроченными хвостами.

5. Развернуть отчетные виджеты по трудозатратам в `Reports` и `Dashboard`.
   Срезы: сотрудник, команда, проект, направление, период.

## Критерии приемки

- Пользователь понимает, что именно мешает закрыть день, неделю и месяц.
- Трудозатраты вводятся единообразно вне зависимости от типа работы.
- Руководитель видит расхождения и неполные данные на фронте.
