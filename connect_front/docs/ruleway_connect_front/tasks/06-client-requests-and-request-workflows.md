# Задания По Блоку: Клиентские Обращения, Заявки И Согласования

## Цель

Собрать фронтовый кейс-менеджмент для клиентской работы: от первого обращения до заявки, согласования и результата.

## Задания

1. Построить единый request workflow map.
   Модули: `HelpDesk`, `Orders`, `RequestApprovals`, `Directories`.

2. Сделать общий timeline клиентского кейса.
   Должны входить: обращения, заявки, встречи, комментарии, документы, согласования.

3. Унифицировать статусные экраны и статусные бейджи.
   Один и тот же статусный язык должен использоваться во всех связанных модулях.

4. Добавить сценарии переходов между сущностями.
   Примеры: обращение -> задача, обращение -> встреча, обращение -> заявка, заявка -> согласование.

5. Расширить клиентский контекст в карточках и списках.
   Нужно: объектные связи, SLA/приоритет, владелец, история, риски, связанные действия.

6. Добавить руководительские фильтры по клиентским кейсам.
   Срезы: по очередям, просрочке, этапу, исполнителю, клиенту, направлению.

## Критерии приемки

- Пользователь может пройти путь клиентского кейса без переключения между несвязанными экранами.
- История обработки прозрачна и собирается в одном месте.
- Из интерфейса понятно, кто отвечает за следующий шаг и на каком этапе находится кейс.
