# Управленческий Контур, Пульс Дня И ИИ-Сводки

## Разделы регламента

- `11. Роль руководителя подразделения и руководителя направления`
- `12. Пульс дня и управленческие сигналы`
- `18. Ежедневная ИИ-сводка по итогам дня`

## Смысл блока

Руководитель должен видеть состояние команды и направления не вручную, а через управленческий контур: пульс дня, сигналы, сводки, контроль зависаний, загрузки и проблемных зон.

## Связанные фронтовые модули

- `Dashboard`
- `WorkPlan`
- `Reports`
- `Notifications`
- `AIAssistant`
- `Profiler`

## Что уже видно во фронте

- Есть дашбордный слой.
- Есть `Reports` и AI-consolidation в `WorkPlan`.
- Есть `Notifications`.
- Есть `AIAssistant`, который уже умеет работать с задачами, встречами и отчетами.

## Чего не хватает для соответствия регламенту

- Нет отдельного фронтового контура `пульс дня` для сотрудника и руководителя.
- Нет единой ленты управленческих сигналов с приоритезацией и подтверждением обработки.
- Ежедневная ИИ-сводка не выглядит как обязательный регулярный продуктовый экран.
- Контроль команды и направления разнесен по разным экранам и не собран в manager cockpit.

## Функциональное направление развития

- Сделать manager cockpit с фокусом на отклонения, хвосты, перегрузки, риски и сигналы.
- Ввести сценарии заполнения и просмотра `пульса дня`.
- Вынести ежедневные ИИ-сводки в самостоятельный фронтовый объект со статусами доставки и чтения.
- Добавить explainability для ИИ-выводов: из каких задач, встреч, обращений и трудозатрат собрана сводка.
