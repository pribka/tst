# Трудозатраты, Закрытие Недели И Отчетность

## Разделы регламента

- `8. Ежедневная фиксация трудозатрат`
- `9. Закрытие недели сотрудником`
- `19. Табели и ежемесячная отчетность`
- `22. Роль системы в формировании отчетности`

## Смысл блока

Система должна обеспечивать ежедневную фиксацию трудозатрат, отсутствие хвостов к концу недели и использовать накопленные данные как основание для табелей и производственной отчетности.

## Связанные фронтовые модули

- `WorkPlan`
- `Reports`
- `Dashboard`
- `vue2TaskComponent`
- `HelpDesk`

## Что уже видно во фронте

- Есть `WorkPlan`, который уже используется как точка ежедневной работы.
- Есть `Reports` с шаблонами и дашбордами.
- В `HelpDesk` есть признаки учета трудозатрат на обращениях.

## Чего не хватает для соответствия регламенту

- Не видно единого ежедневного worklog UX, понятного для всех типов работ.
- Нет явного weekly close assistant/checklist.
- Не видно прозрачного reconcile между задачами, трудозатратами и итоговой отчетностью.
- Нет фронтовой диагностики “что мешает закрыть неделю или месяц”.
- Табельный контур и оперативный контур недостаточно связаны на уровне интерфейса.

## Функциональное направление развития

- Ввести единый worklog layer для задач, обращений, встреч и заявок.
- Сделать мастер закрытия недели с проверкой хвостов.
- Добавить фронтовые расхождения: есть задача без трудозатрат, трудозатраты без объекта, незакрытая просрочка, неполная неделя.
- Связать пользовательские weekly/monthly summary экраны с отчетными дашбордами.
