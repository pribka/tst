# Встречи, Звонки И Внутренние Коммуникации

## Разделы регламента

- `13. Встречи`
- `14. Звонки`
- `17. Чаты и коммуникации внутри системы`

## Смысл блока

Все коммуникации должны оставаться в системе, быть связаны с объектами управления и при необходимости превращаться в дальнейшую работу: задачи, намерения, договоренности, follow-up.

## Связанные фронтовые модули

- `vue2MeetingComponent`
- `Calendar`
- `vue2ChatComponent`
- `vue2CommentsComponent`
- `vue2Files`
- `AIAssistant`
- `Notifications`

## Что уже видно во фронте

- Есть полноценный модуль встреч.
- Есть внутренний чат, комментарии и файлы.
- Есть календарь как планировочный слой.
- В AI-ассистенте уже есть сценарии по встречам и извлечению намерений.

## Чего не хватает для соответствия регламенту

- Нет отдельного выраженного фронтового контура для звонков.
- Коммуникации не собраны в единый communication timeline по объекту.
- Не хватает явных действий `создать задачу`, `создать встречу`, `создать заявку` из чата/комментария/созвона как универсального UX.
- Не видно обязательной связки запись/саммари/намерения/следующие действия.
- Нет явной проверки полноты фиксации встречи: участники, запись, итог, задачи, привязка.

## Функциональное направление развития

- Собрать единый UX коммуникации как объекта системы.
- Добавить call log UI и унифицировать его со встречами.
- Дать пользователю timeline коммуникаций по проекту, клиенту и сотруднику.
- Встроить ИИ-разбор коммуникаций в стандартный поток работы, а не как отдельную экзотику.
