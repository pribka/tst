## Смысл блока

RuleWay должен быть не набором разрозненных экранов, а единой цифровой средой, где любое значимое действие оставляет цифровой след и по возможности связано с объектом управления: направлением, проектом, контрактом, контрагентом, подразделением и сотрудником.

## Связанные фронтовые модули

- `Dashboard`
- `Directories`
- `Projects`
- `Groups`
- `Profiler`
- `Documents`
- `Support`
- `UIModules`

## Что уже видно во фронте

- Есть отдельные сущности и их витрины: проекты, команды, оргструктура, клиенты, позиции, направления.
- Есть базовый операционный вход через `Dashboard`.
- Есть профили сотрудников и командный слой через `Profiler`, `Groups`, `Directories`.
- Есть wiki/knowledge слой через `Support`.

## Чего не хватает для соответствия регламенту

- Единой карты объектов управления во фронте нет: пользователю трудно видеть, какой объект является главным контекстом работы.
- Не видно обязательности привязки действий к объектам управления на уровне массового UX.
- Нет сквозной визуальной модели цифрового следа: что уже связано с проектом, а что осталось вне контекста.
- Не просматривается единый объектный navigation layer, который одинаково ведет в проект, контрагента, обращение, встречу, заявку и историю.

## Функциональное направление развития

- Ввести единый объектный контекст во всех ключевых формах и карточках.
- Сделать универсальные object chips, breadcrumbs и переходы между связанными сущностями.
- Добавить фронтовые индикаторы полноты цифрового следа: есть ли привязки, комментарии, файлы, коммуникации, трудозатраты.
- Собрать объектную навигацию и историю действий вокруг проекта, контрагента, направления и сотрудника.
