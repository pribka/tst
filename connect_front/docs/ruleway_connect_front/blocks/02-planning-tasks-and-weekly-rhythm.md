# Планирование, Задачи И Недельный Ритм

## Разделы регламента

- `6. Горизонт планирования и ритм работы сотрудника`
- `7. Правила работы сотрудника в течение недели`
- `10. Правила работы с незавершенными задачами и статусом «Висяк»`

## Смысл блока

Сотрудник должен жить в недельном ритме: планировать неделю или обозримый горизонт, вести задачи в системе, поддерживать их актуальность и явно обрабатывать незавершенные задачи и висяки.

## Связанные фронтовые модули

- `WorkPlan`
- `vue2TaskComponent`
- `Projects`
- `Calendar`
- `Notifications`
- `AIAssistant`

## Что уже видно во фронте

- Есть `WorkPlan` как центральная точка планирования.
- Есть модуль задач `vue2TaskComponent`.
- Есть календарный слой.
- Есть отдельный AI-экран `workplan-ai-consolidation`.

## Чего не хватает для соответствия регламенту

- Недельный цикл не оформлен как единый продуктовый сценарий от планирования до закрытия.
- Не видно фронтового контура для обязательной причины переноса, зависания и просрочки.
- Статус `висяк` не выглядит как отдельный управляемый UX-сценарий с фильтрами, причинами и контролем.
- Нет явного morning planning flow: пользователь не получает понятный путь “спланируй неделю/день”.
- Нет визуального контроля полноты плана и качества исполнения по неделе.

## Функциональное направление развития

- Собрать weekly workspace: план, календарь, backlog, просрочка, висяки, завершение недели.
- Сделать обязательные reason-capture сценарии для переноса, заморозки, блокера и висяка.
- Добавить виджеты качества недели: без плана, без срока, без привязки, просрочено, зависло.
- Встроить AI-помощь в разбор хвостов и перепланирование недели.
